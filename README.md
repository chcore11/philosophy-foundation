# Philosophy Foundation

一个从真实问题出发，用判断指导行动，再用现实修正判断的个人认知项目。

## 这个项目做什么

这是我的个人“认知基本盘”项目。

它不以收集哲学知识为目标，而是整理我在真实问题中形成的判断：焦虑、选择、行动、欲望、群体、权力、价值排序、亲密关系和人生方向。我要借这些问题训练判断力、价值排序、自我稳定性和行动能力。

## 为什么做

我在做选择时还没有稳定的判断标准，容易依赖钱、流量、绩点、竞赛结果和别人已经取得的成果，判断一件事是否值得做。看到同龄人在保研、考研、就业、自媒体、AI 项目等方向上有进展时，我也容易动摇。

现实不会等我把所有问题想清楚。我需要的不是一套永远正确的答案，而是一套能让我在压力和不确定中继续判断、行动、复盘的系统。

## 这个项目不是什么

- 不是情绪垃圾桶。
- 不是名言摘抄。
- 不是哲学术语堆砌。
- 不是替代行动的精神自慰。
- 不是为了证明自己比别人清醒。

情绪可以成为问题来源，但不能成为文章终点；哲学概念可以帮助辨析，但不能代替自己的判断。

## 项目原则

1. 每篇内容从现实问题出发，保留问题发生的处境。
2. 先区分事实、情绪和解释，再形成暂时判断。
3. 不把复杂问题压成一句漂亮口号。
4. 不拿自己的价值观粗暴审判别人，也不把判断权交给大众。
5. 观点必须接受现实修正，文章允许在复盘后更新。
6. **每一篇哲学笔记，最后都必须回答：这件事如何改变我的现实行动？**
7. 不能用“我在思考”替代行动。

```text
.
├── README.md       # 项目定位、原则和使用方式

├── ROADMAP.md      # 阶段目标、完成标准和近期优先级

├── frameworks/     # 可以直接用于行动和复盘的方法
└── logs/           # 有日期的现实记录、实践结果和阶段回顾
```

四类内容不是按篇幅区分，而是按用途区分：

- `essays/` 回答“我现在如何理解这个问题”。
- `notes/` 保留“这个观点对现实有什么启发和边界”。
- `frameworks/` 回答“下次遇到类似问题，具体怎么做”。
- `logs/` 记录“我是否真的做了，结果怎样”。

## 如何使用

这个项目的基本循环是：

```text
现实问题 → 写清情绪与初始判断 → 重新辨析 → 形成暂时结论
        → 采取一个具体行动 → 记录结果 → 修正原有判断
```

### 遇到新问题时

1. 先记录发生了什么，不急着写成完整文章。
2. 使用 [每篇哲学笔记的行动模板](frameworks/每篇哲学笔记的行动模板.md)，区分问题、情绪和判断。
3. 从暂时结论中提取一个可以执行、可以验证的动作。
4. 给行动设置复盘时间，不用一次思考追求终局答案。

### 整理已有观点时

1. 检查是否保留了现实问题来源。
2. 检查是否把复杂问题写成了方便传播的口号。
3. 检查结论是否说明适用条件、代价或边界。
4. 检查文章最后是否有现实行动和待复盘事项。

### 定期复盘时

不只统计写了多少文章，重点检查：

- 哪个判断真的改变了选择或行为？
- 哪个原则说得通，但现实中做不到？
- 哪个问题反复出现，需要形成框架？
- 哪篇文章已经被现实修正，需要更新？

## 初始学习路线

这条路线来自项目建立时的设想，继续保留，但是否学习、何时学习，要由现实问题决定：

1. 现实问题清单 +《天道》片段：激发问题意识，但不把电视剧当真理
2. 逻辑学基础：训练判断，防止被话术和概念带跑
3. 斯多葛主义：区分可控与不可控，稳定情绪
4. 实用主义：把想法落实到行动和结果
5. 伦理学基础：防止变得冷漠、功利、傲慢
